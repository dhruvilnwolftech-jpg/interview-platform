================================================================================
ARCHITECTURE SUMMARY
================================================================================

SYSTEM DESIGN:
  - 3-tier architecture (Frontend → Backend → Database)
  - WebSocket-based real-time communication
  - Room-based message broadcasting
  - Event-driven synchronization
  - SQLAlchemy ORM abstraction

COMPONENTS:
  - User APK: Code entry + Transparent document
  - Support APK: Session creation + Document management
  - Admin APK: Monitoring dashboard
  - Backend: Flask REST API + Socket.IO
  - Database: SQLite with 4 tables

DATA FLOW:
  1. User → Code verification → Backend
  2. Backend → Session creation → Database
  3. User joins → WebSocket connection
  4. All users → Subscribe to room
  5. Edit → Broadcast to room → All users see update
  6. Changes → Recorded in database → Permanent history
